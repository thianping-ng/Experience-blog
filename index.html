Upon performing the sql delete query for the delete functionality on the website, the system received an error during runtime.

The error stated that I've tried to perform a delete operation or update on "employee" table which is the parent table that has a foreign key relationship with another tabled called "record which is the refernce in relationship with employee table, which resulted in a foreign key constraint. Due to existing record in the database

So in order for me to write a delete function, I've create a Delete class in the employee controller which takes in ID as a parameter. So Delete class will store a list of employee variable called emp which retrieves from GetEmployeeData in the data access layer, and upon confirmation of deleting a specific record of the employee. The validate anti forgery token ensure that only those specific individual can perform such action like deleting a employee info from the database.

So in the data access layer, I've created a class called Delete wich return number of rows that updates or delete records of info from the database.

The first "query1" delete a employee info from the "record"table by their employee ID, where the second query "query2" delete a employee info from the employee table based on their employee ID.

The reason why I had to write two query, is that due to foreign key constraint in the database where just by deleting a record it may affect the employee table as it has a foreign key "employee ID" with the record table. In addtion, after performing two query transaction variable "rowAffected" return the number of updated or deleted rows.

In addition, ubuntu server is a version of ubuntu operating system is lightweight and efficient runs fast, able to run apps quicker it act as a backbone for the server

aside from the normal traffic, a secure channel which reside in the secure route to send request over the network.
